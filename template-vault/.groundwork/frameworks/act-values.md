# ACT Values Clarification

ACT distinguishes values from goals: a value is a **chosen direction** you want to keep moving in; a goal is an outcome you achieve and tick off. You can't arrive at a value. You can only stay aligned with it or drift from it.

Use this framework as the primary opening for `groundwork-session values`, especially for seeded/empty artifacts. Generic "what's on your mind" kickstarts surface context but not values; ACT questions surface values directly from lived experience.

## How to run

- Open with `peak-moment` or `domain-pick` — pick based on user energy. If user sounds reflective/mellow, peak-moment. If user sounds restless/in-action, domain-pick.
- Follow user's concrete content, not abstract labels. When the user says a word like "freedom" or "growth", immediately ask for a behavior or episode that shows it.
- Use `frustration-reveal` and `behavior-test` to check declared values against lived ones. Contradictions are gold — log them in the session note.
- Close with `direction-not-goal` to make at least one value concrete and movable in the next week.

## Pairings

- After ACT opens a domain, 5-Prism deepens the emotional layer.
- Ikigai (`love`, `good-at`) pairs well with `no-payoff` and `free-hour` — both triangulate what the user gravitates toward when freed from obligation.
- `gleb-modes` activates if user hits avoidance or fractal overwhelm during exploration.
